Authentication API Documentation

Overview

Welcome to the Authentication API documentation! This API is crafted with Node.js and Express to provide robust user authentication features for web applications. Following the MVC (Model-View-Controller) pattern, it ensures a secure and scalable authentication system.

Features

  • User Registration: Create a new account with ease.
  • User Authentication: Log in securely with your credentials.
  • Logout: Terminate your current session effortlessly.
  • Profile Management: Update and retrieve user profile information effortlessly.
  • Home Route: Quickly navigate to the home page.

API Endpoints

  • POST /api/users: Register a new user.
  • POST /api/users/auth: Authenticate a user.
  • POST /api/users/logout: Logout a user.
  • GET /api/users/profile: Get user profile information.
  • PUT /api/users/profile: Update user profile information.
  • GET /: Home route.

Functionality

This API provides a range of functionalities for user authentication:

  • Register: Users can create new accounts with required information.
  • Login: Securely log in with provided credentials.
  • Logout: Terminate the current session.
  • Update Profile: Modify user profile information.
  • Get User Profile: Retrieve user profile information.
  • Home: Navigate to the home page.

Advanced Features

  • MongoDB Integration: Securely store user information using MongoDB.
  • Custom Routing Models: Implemented for handling different routes and actions.
  • Secure JWT Token: Use JWT for authentication with tokens valid for one month.
  • HTTP-only Cookie: Enhance security with HTTP-only cookies, protecting against XSS, XSRF, and cookie hijacking.
  • Asynchronous Error Handling: Efficient error handling for a seamless user experience.
  • Environment Variables: Utilize a separate .env file for storing sensitive information.

Integration with Other Projects

Easily integrate this authentication project with other web applications or projects requiring user authentication. Save time and effort by incorporating the provided authentication functionalities.

Security Measures

Designed with security in mind, this project includes measures against common vulnerabilities such as XSS, XSRF, and cookie hijacking. Utilizing JWT tokens, HTTP-only cookies, and secure error handling ensures the security and integrity of user data.

Note: While this project includes security measures, regular updates, best practices, and security audits are essential for maintaining a robust and secure authentication system.

Installation and Setup

Follow these steps to set up the project:

  1. Clone the repository to your local machine.
  2. Install required dependencies using npm or yarn: npm install or yarn install.
  3. Create a .env file in the root directory and set necessary environment variables.
  4. Start the application: npm start or yarn start.
  5. Access the application on the specified port (default is 3000): http://localhost:3000.

Sample .env File

NODE_ENV=development
PORT=6000
MONGO_URI=''
JWT_SECRET_KEY=abc123
